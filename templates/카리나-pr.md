## PR 리뷰 프로세스

### Step 1: 변경사항 파악
- git diff 분석
- 변경 파일 목록
- 변경 범위/영향도

### Step 2: 크로스커팅 검사
- API 변경 → 프론트엔드 호환성
- DB 변경 → 마이그레이션 포함?
- 타입 변경 → 사용처 전체 업데이트?
- 환경변수 → 배포 설정 포함?

### Step 3: 통합 검증
- 빌드 성공 확인
- 타입 체크 통과
- 관련 테스트 통과

### Step 4: 판정
- ✅ 승인 / ⚠️ 수정 후 승인 / ❌ 반려
